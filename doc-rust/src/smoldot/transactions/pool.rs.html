18//! General-purpose transactions pool.
19//!
20//! The transactions pool is a complex data structure that holds a list of pending transactions,
21//! in other words transactions that should later be included in blocks, and a list of
22//! transactions that have been included in non-finalized blocks.
23//!
24//! See the [parent module's documentation](..) for an overview of transactions.
25//!
26//! # Overview
27//!
28//! The transactions pool stores a list of transactions that the local node desires to include in
29//! blocks, and a list of transactions that have already been included in blocks. Each of these
30//! transactions is either validated or not. A transaction in a block is assumed to always succeed
31//! validation. A validated transaction that isn't present in any block is a transaction that is
32//! assumed to be includable in a block in the future.
33//!
34//! The order in which transactions can be included in a block follows a complex system of
35//! "provided" and "required" tags. A transaction that *requires* some tags can only be included
36//! after all these tags have been *provided* by transactions earlier in the chain.
37//!
38//! The transactions pool isn't only about deciding which transactions to include in a block when
39//! authoring, but also about tracking the status of interesting transactions between the moment
40//! they become interesting and the moment the block they are included in becomes finalized. This
41//! is relevant both if the local node can potentially author blocks or not.
42//!
43//! The transactions pool tracks the height of the *best* chain, and only of the best chain. More
44//! precisely, it is aware of the height of the current best block. Forks are tracked.
45//!
46//! # Usage
47//!
48//! A [`Pool`] is a collection of transactions. Each transaction in the pool exposes three
49//! properties:
50//!
51//! - Whether or not it has been validated, and if yes, the block against which it has been
52//! validated and the characteristics of the transaction (as provided by the runtime). These
53//! characterstics are: the tags it provides and requires, its longevity, and its priority.
54//! See [the `validate` module](../validate) for more information.
55//! - The height of the block, if any, in which the transaction has been included.
56//! - A so-called user data, an opaque field controlled by the API user.
57//!
58//! Use [`Pool::add_unvalidated`] to add to the pool a transaction that should be included in a
59//! block at a later point in time.
60//!
61//! When a new block is considered as best, use [`Pool::retract_blocks`] to remove all the re-orged
62//! blocks, then [`Pool::append_empty_block`] and
63//! [`Pool::best_block_add_transaction_by_scale_encoding`] to add the new block(s). The
64//! transactions that are passed to [`Pool::best_block_add_transaction_by_scale_encoding`] are
65//! added to the pool.
66//!
67//! Use [`Pool::unvalidated_transactions`] to obtain the list of transactions that should be
68//! validated. Validation should be performed using the [`validate`](../validate) module, and
69//! the result reported with [`Pool::set_validation_result`].
70//!
71//! Use [`Pool::remove_included`] when a block has been finalized to remove from the pool the
72//! transactions that are present in the finalized block and below.
73//!
74//! When authoring a block, use [`Pool::append_empty_block`] and
75//! [`Pool::best_block_includable_transactions`] to determine which transaction to include
76//! next. Use [`Pool::best_block_add_transaction_by_id`] when the transaction has been included.
77//!
78//! # Out of scope
79//!
80//! The following are examples of things that are related transactions pool to but out of scope of
81//! this data structure:
82//!
83//! - Watching the state of transactions.
84//! - Sending transactions to other peers.
85//!

98/// Identifier of a transaction stored within the [`Pool`].
99///
100/// Identifiers can be re-used by the pool. In other words, a transaction id can compare equal to
101/// an older transaction id that is no longer in the pool.
102//
103// Implementation note: corresponds to indices within [`Pool::transactions`].
104#[derive(Debug, Clone, Copy, PartialEq, Eq, PartialOrd, Ord, Hash)]
105pub struct TransactionId(usize);
106
107/// Configuration for [`Pool::new`].
108pub struct Config {
109    /// Number of transactions to initially allocate memory for.
110    ///
111    /// > **Note**: This should take into account the fact that the pool will contain the
112    /// >           transactions included in new blocks. In other words, it should be equal to
113    /// >           `expected_max_reorg_depth * expected_max_transactions_per_block +
114    /// >           max_concurrent_desired_transactions`.
115    pub capacity: usize,
116
117    /// Height of the finalized block at initialization.
118    ///
119    /// The [`Pool`] doesn't track which block is finalized. This value is only used to initialize
120    /// the best block number. The field could also have been called `best_block_height`, but doing
121    /// so might created confusion.
122    ///
123    /// Non-finalized blocks should be added to the pool after initialization using
124    /// [`Pool::append_empty_block`].
125    pub finalized_block_height: u64,
126
127    /// Seed for randomness used to avoid HashDoS attacks.
128    pub randomness_seed: [u8; 16],
129}

131/// Data structure containing transactions. See the module-level documentation for more info.
132pub struct Pool<TTx> {
133    /// Actual list of transactions.
134    transactions: slab::Slab<Transaction<TTx>>,
135
136    /// List of transactions (represented as indices within [`Pool::transactions`]) whose status
137    /// is "not validated".
138    // TODO: shrink_to_fit from time to time?
139    not_validated: HashSet<TransactionId, fnv::FnvBuildHasher>,
140
141    /// Transaction ids (i.e. indices within [`Pool::transactions`]) indexed by the BLAKE2 hash
142    /// of the bytes of the transaction.
143    by_hash: BTreeSet<([u8; 32], TransactionId)>,
144
145    /// Transaction ids (i.e. indices within [`Pool::transactions`]) indexed by the block height
146    /// in which the transaction is included.
147    by_height: BTreeSet<(u64, TransactionId)>,
148
149    /// Validated transaction ids (i.e. indices within [`Pool::transactions`]) that are includable
150    /// in the chain, indexed by the priority value provided by the validation.
151    includable: BTreeSet<(u64, TransactionId)>,
152
153    /// Validated transaction ids (i.e. indices within [`Pool::transactions`]) indexed by the
154    /// block height at which their validation expires.
155    by_validation_expiration: BTreeSet<(u64, TransactionId)>,
156
157    /// List of all tags that are in the `provides` or `requires` tag lists of any of the validated
158    /// transactions.
159    // TODO: shrink_to_fit from time to time?
160    tags: hashbrown::HashMap<Vec<u8>, TagInfo, crate::util::SipHasherBuild>,
161
162    /// Height of the latest best block, as known from the pool.
163    best_block_height: u64,
164}

166/// Entry in [`Pool::transactions`].
167struct Transaction<TTx> {
168    /// Bytes corresponding to the SCALE-encoded transaction.
169    scale_encoded: Vec<u8>,
170
171    /// If `Some`, contains the outcome of the validation of this transaction and the block height
172    /// it was validated against.
173    validation: Option<(u64, ValidTransaction)>,
174
175    /// If `Some`, the height of the block at which the transaction has been included.
176    included_block_height: Option<u64>,
177
178    /// User data chosen by the user.
179    user_data: TTx,
180}

182/// Entry in [`Pool::tags`].
183struct TagInfo {
184    /// List of validated transactions that have the tag in their `provides` tags list.
185    // TODO: shrink_to_fit from time to time?
186    provided_by: hashbrown::HashSet<TransactionId, fnv::FnvBuildHasher>,
187
188    /// List of validated transactions that have the tag in their `requires` tags list.
189    // TODO: shrink_to_fit from time to time?
190    required_by: hashbrown::HashSet<TransactionId, fnv::FnvBuildHasher>,
191
192    /// Number of transactions in [`TagInfo::provided_by`] that are included in the chain.
193    ///
194    /// Note that a value strictly superior to 1 indicates some kind of bug in the logic of the
195    /// runtime. However, we don't care about this in the pool and just want the pool to function
196    /// properly.
197    known_to_be_included_in_chain: usize,
198}

228    /// Inserts a new non-validated transaction in the pool.
229    pub fn add_unvalidated(&mut self, scale_encoded: Vec<u8>, user_data: TTx) -> TransactionId {
230        self.add_unvalidated_inner(scale_encoded, None, user_data)
231    }

233    /// Inserts a new non-validated transaction in the pool.
234    fn add_unvalidated_inner(
235        &mut self,
236        scale_encoded: impl AsRef<[u8]> + Into<Vec<u8>>,
237        included_block_height: Option<u64>,
238        user_data: TTx,
239    ) -> TransactionId {
240        let hash = blake2_hash(scale_encoded.as_ref());
241
242        let tx_id = TransactionId(self.transactions.insert(Transaction {
243            scale_encoded: scale_encoded.into(),
244            validation: None,
245            included_block_height,
246            user_data,
247        }));
248
249        let _was_inserted = self.by_hash.insert((hash, tx_id));
250        debug_assert!(_was_inserted);
251
252        let _was_inserted = self.not_validated.insert(tx_id);
253        debug_assert!(_was_inserted);
254
255        if let Some(included_block_height) = included_block_height {
256            let _was_inserted = self.by_height.insert((included_block_height, tx_id));
257            debug_assert!(_was_inserted);
258        }
259
260        tx_id
261    }

263    /// Removes from the pool the transaction with the given identifier.
264    ///
265    /// # Panic
266    ///
267    /// Panics if the identifier is invalid.
268    ///
269    #[track_caller]
270    pub fn remove(&mut self, id: TransactionId) -> TTx {
271        self.unvalidate_transaction(id);
272        let _removed = self.not_validated.remove(&id);
273        debug_assert!(_removed);
274
275        let tx = self.transactions.remove(id.0);
276
277        if let Some(included_block_height) = tx.included_block_height {
278            let _removed = self.by_height.remove(&(included_block_height, id));
279            debug_assert!(_removed);
280        }
281
282        let _removed = self.by_hash.remove(&(blake2_hash(&tx.scale_encoded), id));
283        debug_assert!(_removed);
284
285        tx.user_data
286    }

288    /// Removes from the pool all the transactions that are included in a block whose height is
289    /// inferior or equal to the one passed as parameter.
290    ///
291    /// Use this method when a block has been finalized.
292    ///
293    /// The returned iterator is guaranteed to remove all transactions even if it is dropped
294    /// eagerly.
295    pub fn remove_included(
296        &mut self,
297        block_inferior_of_equal: u64,
298    ) -> impl Iterator<Item = (TransactionId, TTx)> {
299        // First, unvalidate all the transactions that we are going to remove.
300        // This is done separately ahead of time in order to guarantee that there is no state
301        // mismatch when `unvalidate_transaction` is entered.
302        for tx_id in self
303            .by_height
304            .range(
305                (0, TransactionId(usize::MIN))
306                    ..=(block_inferior_of_equal, TransactionId(usize::MAX)),
307            )
308            .map(|(_, id)| *id)
309            .collect::<Vec<_>>()
310        {
311            self.unvalidate_transaction(tx_id);
312        }
313
314        // Extracts all the transactions that we are about to remove from `by_height`.
315        let to_remove = {
316            let remaining_txs = self
317                .by_height
318                .split_off(&(block_inferior_of_equal + 1, TransactionId(usize::MIN)));
319            mem::replace(&mut self.by_height, remaining_txs)
320        };

322        struct ToRemoveIterator<'a, TTx> {
323            pool: &'a mut Pool<TTx>,
324            transactions: btree_set::IntoIter<(u64, TransactionId)>,
325        }
326
327        impl<'a, TTx> Iterator for ToRemoveIterator<'a, TTx>
328        where
329            // `FusedIterator` is necessary in order for the `Drop` implementation to not panic.
330            btree_set::IntoIter<(u64, TransactionId)>: iter::FusedIterator,
331        {
332            type Item = (TransactionId, TTx);
333
334            fn next(&mut self) -> Option<Self::Item> {
335                let (_height, tx_id) = self.transactions.next()?;
336
337                let tx = self.pool.transactions.remove(tx_id.0);
338                debug_assert!(tx.validation.is_none());
339                debug_assert_eq!(tx.included_block_height, Some(_height));
340
341                let _removed = self
342                    .pool
343                    .by_hash
344                    .remove(&(blake2_hash(&tx.scale_encoded), tx_id));
345                debug_assert!(_removed);
346
347                Some((tx_id, tx.user_data))
348            }
349
350            fn size_hint(&self) -> (usize, Option<usize>) {
351                self.transactions.size_hint()
352            }
353        }
354
355        impl<'a, TTx> ExactSizeIterator for ToRemoveIterator<'a, TTx> where
356            btree_set::IntoIter<(u64, TransactionId)>: ExactSizeIterator
357        {
358        }
359
360        impl<'a, TTx> Drop for ToRemoveIterator<'a, TTx> {
361            fn drop(&mut self) {
362                // Drain the rest of the iterator in order to remove the transactions even if
363                // the iterator is dropped early.
364                while self.next().is_some() {}
365            }
366        }
367
368        ToRemoveIterator {
369            pool: self,
370            transactions: to_remove.into_iter(),
371        }
372    }

374    /// Returns a list of transactions whose state is "not validated", their user data, and the
375    /// height of the block they should be validated against.
376    ///
377    /// The block height a transaction should be validated against is always equal to either the
378    /// block at which it has been included minus one, or the current best block. It is yielded by
379    /// the iterator for convenience, to avoid writing error-prone code.
380    pub fn unvalidated_transactions(
381        &self,
382    ) -> impl ExactSizeIterator<Item = (TransactionId, &TTx, u64)> {
383        self.not_validated.iter().copied().map(move |tx_id| {
384            let tx = self.transactions.get(tx_id.0).unwrap();
385            let height = tx
386                .included_block_height
387                .map(|n| n.checked_sub(1).unwrap())
388                .unwrap_or(self.best_block_height);
389            (tx_id, &tx.user_data, height)
390        })
391    }

426    /// Finds the transactions in the pool whose bytes are `scale_encoded`.
427    ///
428    /// This operation has a complexity of `O(log n)` where `n` is the number of entries in the
429    /// pool.
430    pub fn transactions_by_scale_encoding(
431        &self,
432        scale_encoded: &[u8],
433    ) -> impl Iterator<Item = TransactionId> {
434        let hash = blake2_hash(scale_encoded);
435        self.by_hash
436            .range((hash, TransactionId(usize::MIN))..=(hash, TransactionId(usize::MAX)))
437            .map(|(_, tx_id)| *tx_id)
438    }

440    /// Returns the best block height according to the pool.
441    ///
442    /// This initially corresponds to the value in [`Config::finalized_block_height`], is
443    /// incremented by one every time [`Pool::append_empty_block`], and is decreased when

449    /// Adds a block to the chain tracked by the transactions pool.
450    pub fn append_empty_block(&mut self) {
451        self.best_block_height = self.best_block_height.checked_add(1).unwrap();
452
453        // Un-validate the transactions whose validation longevity has expired.
454        for tx_id in self
455            .by_validation_expiration
456            .range(
457                (0, TransactionId(usize::MIN))
458                    ..=(self.best_block_height, TransactionId(usize::MAX)),
459            )
460            .map(|(_, id)| *id)
461            .collect::<Vec<_>>()
462        {
463            self.unvalidate_transaction(tx_id);
464        }
465    }

467    /// Pop a certain number of blocks from the list of blocks.
468    ///
469    /// Transactions that were included in these blocks remain in the transactions pool.
470    ///
471    /// Returns the list of transactions that were in blocks that have been retracted, with the
472    /// height of the block at which they were.
473    ///
474    /// # Panic
475    ///
476    /// Panics if `num_to_retract > self.best_block_height()`, in other words if the block number
477    /// would go in the negative.
478    ///
479    pub fn retract_blocks(
480        &mut self,
481        num_to_retract: u64,
482    ) -> impl Iterator<Item = (TransactionId, u64)> {
483        // Checks that there's no transaction included above `self.best_block_height`.
484        debug_assert!(
485            self.by_height
486                .range((self.best_block_height + 1, TransactionId(usize::MIN),)..,)
487                .next()
488                .is_none()
489        );
490
491        // Update `best_block_height` as first step, in order to panic sooner in case of underflow.
492        self.best_block_height = self.best_block_height.checked_sub(num_to_retract).unwrap();
493
494        // List of transactions that were included in these blocks.
495        let transactions_to_retract = self
496            .by_height
497            .range((self.best_block_height + 1, TransactionId(usize::MIN))..)
498            .map(|(block_height, tx_id)| (*tx_id, *block_height))
499            .collect::<Vec<_>>();
500
501        // Set `included_block_height` to `None` for each of them.
502        for (transaction_id, _) in &transactions_to_retract {
503            self.unvalidate_transaction(*transaction_id);
504
505            let tx_data = self.transactions.get_mut(transaction_id.0).unwrap();
506            debug_assert!(tx_data.included_block_height.unwrap() > self.best_block_height);
507            self.by_height
508                .remove(&(tx_data.included_block_height.unwrap(), *transaction_id));
509            tx_data.included_block_height = None;
510        }
511
512        // Return retracted transactions from highest block to lowest block.
513        transactions_to_retract.into_iter().rev()
514    }

516    /// Returns all the transactions that can be included in the highest block.
517    ///
518    /// Use this function if you are currently authoring a block.
519    ///
520    /// The transactions are returned by decreasing priority. Re-ordering the transactions might
521    /// lead to the runtime returning errors. It is safe, however, to skip some transactions
522    /// altogether if desired.
523    pub fn best_block_includable_transactions(
524        &self,
525    ) -> impl Iterator<Item = (TransactionId, &TTx)> {
526        self.includable
527            .iter()
528            .rev()
529            .map(|(_, tx_id)| (*tx_id, &self.transactions[tx_id.0].user_data))
530    }

532    /// Adds a single-SCALE-encoded transaction to the highest block.
533    ///
534    /// The transaction is compared against the list of non-included transactions that are already
535    /// in the pool. If a non-included transaction with the same bytes is found, it is switched to
536    /// the "included" state and [`AppendBlockTransaction::NonIncludedUpdated`] is returned.
537    /// Otherwise, [`AppendBlockTransaction::Unknown`] is returned and the transaction can be
538    /// inserted in the pool.
539    ///
540    /// > **Note**: This function is equivalent to calling
541    /// >           [`Pool::transactions_by_scale_encoding`], removing the transactions that are
542    /// >           already included (see [`Pool::included_block_height`]), then calling
543    /// >           [`Pool::best_block_add_transaction_by_id`] with one of the transactions
544    /// >           that remain.
545    pub fn best_block_add_transaction_by_scale_encoding<'a, 'b>(
546        &'a mut self,
547        bytes: &'b [u8],
548    ) -> AppendBlockTransaction<'a, 'b, TTx> {
549        let non_included = {
550            let hash = blake2_hash(bytes);
551            self.by_hash
552                .range((hash, TransactionId(usize::MIN))..=(hash, TransactionId(usize::MAX)))
553                .find(|(_, tx_id)| {
554                    self.transactions
555                        .get(tx_id.0)
556                        .unwrap()
557                        .included_block_height
558                        .is_none()
559                })
560                .map(|(_, tx_id)| *tx_id)
561        };
562
563        if let Some(tx_id) = non_included {
564            debug_assert_eq!(self.transactions[tx_id.0].scale_encoded, bytes);
565            self.best_block_add_transaction_by_id(tx_id);
566            AppendBlockTransaction::NonIncludedUpdated {
567                id: tx_id,
568                user_data: &mut self.transactions[tx_id.0].user_data,
569            }
570        } else {
571            AppendBlockTransaction::Unknown(Vacant { inner: self, bytes })
572        }
573    }

575    /// Adds a transaction to the block being appended.
576    ///
577    /// # Panic
578    ///
579    /// Panics if the transaction with the given id is invalid.
580    /// Panics if the transaction with the given id was already included in the chain.
581    ///
582    pub fn best_block_add_transaction_by_id(&mut self, id: TransactionId) {
583        // Sanity check.
584        assert!(self.transactions[id.0].included_block_height.is_none());
585
586        // We can in principle always discard the current validation status of the transaction.
587        // However, if the transaction has been validated against the parent of the block, we want
588        // to keep this validation status as an optimization.
589        // Since updating the status of a transaction is a rather complicated state change, the
590        // approach taken here is to always un-validate the transaction then re-validate it.
591        let revalidation = if let Some(validation) = self.transactions[id.0].validation.as_ref() {
592            if validation.0 + 1 == self.best_block_height {
593                Some(validation.clone())
594            } else {
595                None
596            }
597        } else {
598            None
599        };
600        self.unvalidate_transaction(id);
601
602        // Mark the transaction as included.
603        self.transactions[id.0].included_block_height = Some(self.best_block_height);
604        self.by_height.insert((self.best_block_height, id));
605
606        // Re-set the validation status of the transaction that was extracted earlier.
607        if let Some((block_number_validated_against, result)) = revalidation {
608            self.set_validation_result(id, block_number_validated_against, result);
609        }
610    }

612    /// Sets the outcome of validating the transaction with the given identifier.
613    ///
614    /// The block number must be the block number against which the transaction has been
615    /// validated.
616    ///
617    /// Has no effect if the transaction has been included in the chain and the validation has
618    /// been performed against a block other than the parent of the block in which it was included.
619    ///
620    /// Has no effect if the transaction has already been validated against the same or a higher
621    /// block.
622    ///
623    /// > **Note**: If the transaction validation fails, use [`Pool::remove`] to remove the
624    /// >           transaction instead. Invalid transactions stay invalid forever and thus aren't
625    /// >           meant to be left in the pool.
626    ///
627    /// # Panic
628    ///
629    /// Panics if the transaction with the given id is invalid.
630    ///
631    pub fn set_validation_result(
632        &mut self,
633        id: TransactionId,
634        block_number_validated_against: u64,
635        result: ValidTransaction,
636    ) {
637        // If the transaction has been included in a block, immediately return if the validation
638        // has been performed against a different block.
639        if self
640            .transactions
641            .get(id.0)
642            .unwrap()
643            .included_block_height
644            .map_or(false, |b| b != block_number_validated_against + 1)
645        {
646            return;
647        }
648
649        // Immediately return if the transaction has been validated against a better block.
650        if self
651            .transactions
652            .get(id.0)
653            .unwrap()
654            .validation
655            .as_ref()
656            .map_or(false, |(b, _)| *b >= block_number_validated_against)
657        {
658            return;
659        }
660
661        // If the transaction was already validated, we don't try to update all the fields of
662        // `self` as it would be rather complicated. Instead we mark the transaction as not
663        // validated then mark it again as validated.
664        self.unvalidate_transaction(id);
665        debug_assert!(self.transactions[id.0].validation.is_none());
666
667        // Whether the transaction can be included at the head of the chain. Set to `false` below
668        // if there is a reason why not.
669        let mut includable = self.transactions[id.0].included_block_height.is_none();
670
671        for tag in &result.provides {
672            let tag_info = self.tags.entry(tag.clone()).or_insert_with(|| TagInfo {
673                provided_by: Default::default(),
674                required_by: Default::default(),
675                known_to_be_included_in_chain: 0,
676            });
677
678            if self.transactions[id.0].included_block_height.is_some() {
679                tag_info.known_to_be_included_in_chain += 1;
680
681                if tag_info.known_to_be_included_in_chain == 1 {
682                    // All other transactions that provide the same tag are not longer includable.
683                    for other_tx_id in &tag_info.provided_by {
684                        let _was_in = self.includable.remove(&(
685                            self.transactions[other_tx_id.0]
686                                .validation
687                                .as_ref()
688                                .unwrap()
689                                .1
690                                .priority,
691                            *other_tx_id,
692                        ));
693                        debug_assert!(_was_in);
694                    }
695
696                    // All other transactions that require this tag are now includable.
697                    for other_tx_id in &tag_info.required_by {
698                        let _was_inserted = self.includable.insert((
699                            self.transactions[other_tx_id.0]
700                                .validation
701                                .as_ref()
702                                .unwrap()
703                                .1
704                                .priority,
705                            *other_tx_id,
706                        ));
707                        debug_assert!(_was_inserted);
708                    }
709                }
710            }
711
712            if tag_info.known_to_be_included_in_chain >= 1 {
713                includable = false;
714            }
715
716            tag_info.provided_by.insert(id);
717        }
718
719        for tag in &result.requires {
720            let tag_info = self.tags.entry(tag.clone()).or_insert_with(|| TagInfo {
721                provided_by: Default::default(),
722                required_by: Default::default(),
723                known_to_be_included_in_chain: 0,
724            });
725
726            tag_info.required_by.insert(id);
727
728            if tag_info.known_to_be_included_in_chain == 0 {
729                includable = false;
730            }
731        }
732
733        self.by_validation_expiration.insert((
734            block_number_validated_against.saturating_add(result.longevity.get()),
735            id,
736        ));
737
738        if includable {
739            self.includable.insert((result.priority, id));
740        }
741
742        self.transactions[id.0].validation = Some((block_number_validated_against, result));
743    }
744
745    /// Sets a transaction's status to "not validated".
746    ///
747    /// # Panic
748    ///
749    /// Panics if the identifier is invalid.
750    ///
751    fn unvalidate_transaction(&mut self, tx_id: TransactionId) {
752        // No effect if wasn't validated.
753        let Some((block_height_validated_against, validation)) =
754            self.transactions[tx_id.0].validation.take()
755        else {
756            return;
757        };
758
759        // We don't care in this context whether the transaction was includable or not, and we
760        // call `remove` in both cases.
761        self.includable.remove(&(validation.priority, tx_id));
762
763        for tag in validation.provides {
764            let tag_info = self.tags.get_mut(&tag).unwrap();
765
766            let _was_in = tag_info.provided_by.remove(&tx_id);
767            debug_assert!(_was_in);
768
769            if self.transactions[tx_id.0].included_block_height.is_some() {
770                tag_info.known_to_be_included_in_chain -= 1;
771
772                if tag_info.known_to_be_included_in_chain == 0 {
773                    // All other transactions that provide the same tag are now includable.
774                    // Note that in practice they most likely are not, but we prioritize the
775                    // consistency and simplify of the pool implementation rather than trying to
776                    // be smart.
777                    for other_tx_id in &tag_info.provided_by {
778                        let _was_inserted = self.includable.insert((
779                            self.transactions[other_tx_id.0]
780                                .validation
781                                .as_ref()
782                                .unwrap()
783                                .1
784                                .priority,
785                            *other_tx_id,
786                        ));
787                        debug_assert!(_was_inserted);
788                    }
789
790                    // All other transactions that require this tag are no longer includable.
791                    for other_tx_id in &tag_info.required_by {
792                        let _was_in = self.includable.remove(&(
793                            self.transactions[other_tx_id.0]
794                                .validation
795                                .as_ref()
796                                .unwrap()
797                                .1
798                                .priority,
799                            *other_tx_id,
800                        ));
801                        debug_assert!(_was_in);
802                    }
803                }
804            }
805
806            if tag_info.provided_by.is_empty() && tag_info.required_by.is_empty() {
807                self.tags.remove(&tag).unwrap();
808            }
809        }
810
811        for tag in validation.requires {
812            let tag_info = self.tags.get_mut(&tag).unwrap();
813
814            let _was_in = tag_info.required_by.remove(&tx_id);
815            debug_assert!(_was_in);
816
817            if tag_info.provided_by.is_empty() && tag_info.required_by.is_empty() {
818                self.tags.remove(&tag).unwrap();
819            }
820        }
821
822        self.not_validated.insert(tx_id);
823
824        let _was_in = self.by_validation_expiration.remove(&(
825            block_height_validated_against.saturating_add(validation.longevity.get()),
826            tx_id,
827        ));
828        debug_assert!(_was_in);
829    }
830}

858/// See [`Pool::best_block_add_transaction_by_scale_encoding`].
859#[derive(Debug)]
860pub enum AppendBlockTransaction<'a, 'b, TTx> {
861    /// Transaction to add isn't in the list of non-included transactions. It can be added to the
862    /// pool.
863    Unknown(Vacant<'a, 'b, TTx>),
864    /// Transaction to add is present in the list of non-included transactions. It is now
865    /// considered included.
866    NonIncludedUpdated {
867        /// Identifier of the non-included transaction with the same bytes.
868        id: TransactionId,
869        /// User data stored alongside with that transaction.
870        user_data: &'a mut TTx,
871    },
872}
873
874/// See [`AppendBlockTransaction::Unknown`].
875pub struct Vacant<'a, 'b, TTx> {
876    inner: &'a mut Pool<TTx>,
877    bytes: &'b [u8],
878}
879
880impl<'a, 'b, TTx> Vacant<'a, 'b, TTx> {
881    /// Inserts the transaction in the pool.
882    pub fn insert(self, user_data: TTx) -> TransactionId {
883        self.inner
884            .add_unvalidated_inner(self.bytes, Some(self.inner.best_block_height), user_data)
885    }
886}
887
888impl<'a, 'b, TTx: fmt::Debug> fmt::Debug for Vacant<'a, 'b, TTx> {
889    fn fmt(&self, f: &mut fmt::Formatter) -> fmt::Result {
890        fmt::Debug::fmt(&self.inner, f)
891    }
892}
