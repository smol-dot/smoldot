18//! *All-forks* header and body syncing.
19//!
20//! # Overview
21//!
22//! This state machine holds:
23//!
24//! - A list of sources of blocks, maintained by the API user.
25//!  - For each source, a list of blocks hashes known by the source.
26//! - The latest known finalized block.
27//! - A tree of valid non-finalized blocks that all descend from the latest known finalized block.
28//! - (if full mode) A list of block headers whose body is currently being downloaded.
29//! - A list of block header waiting to be verified and whose ancestry with the latest finalized
30//!   block is currently unknown.
31//!
32//! The state machine has the objective to synchronize the tree of non-finalized blocks with its
33//! equivalent on the sources added by the API user.
34//!
35//! Because it is not possible to predict which block in this tree is going to be finalized in
36//! the future, the entire tree needs to be synchronized.
37//!
38//! > **Example**: If the latest finalized block is block number 4, and the tree contains blocks
39//! >              5, 6, and 7, and a source announces a block 5 that is different from the
40//! >              locally-known block 5, a block request will be emitted for this block 5, even
41//! >              if it is certain that this "other" block 5 will not become the local best
42//! >              block. This is necessary in case it is this other block 5 that will end up
43//! >              being finalized.
44//!
45//! # Bounded and unbounded containers
46//!
47//! It is important to limit the memory usage of this state machine no matter how the
48//! potentially-malicious sources behave.
49//!
50//! The state in this state machine can be put into three categories:
51//!
52//! - Each source of blocks has a certain fixed-size state associated to it (containing for
53//!   instance its best block number and height). Each source also has up to one in-flight
54//!   request, which might incur more memory usage. Managing this additional request is out of
55//!   scope of this module. The user of this module is expected to limit the number of
56//!   simultaneous sources.
57//!
58//! - A set of verified blocks that descend from the latest finalized block. This set is
59//!   unbounded. The consensus and finalization algorithms of the chain are supposed to limit
60//!   the number of possible blocks in this set.
61//!
62//! - A set of blocks that can't be verified yet. Receiving a block announce inserts an element
63//!   in this set. In order to handle situations where a malicious source announces lots of
64//!   invalid blocks, this set must be bounded. Once it has reached a certain size, the blocks
65//!   with the highest block number are discarded if their parent is also in this set or being
66//!   downloaded from a source.
67//!
68//! Consequently, and assuming that the number of simultaneous sources is bounded, and that
69//! the consensus and finalization algorithms of the chain are properly configured, malicious
70//! sources can't indefinitely grow the state in this state machine.
71//! Malicious sources, however, can potentially increase the number of block requests required to
72//! download a long fork. This is, at most, an annoyance, and not a vulnerability.
73//!

93/// Configuration for the [`AllForksSync`].
94#[derive(Debug)]
95pub struct Config {
96    /// Information about the latest finalized block and its ancestors.
97    pub chain_information: chain_information::ValidChainInformation,
98
99    /// Number of bytes used when encoding/decoding the block number. Influences how various data
100    /// structures should be parsed.
101    pub block_number_bytes: usize,
102
103    /// If `false`, blocks containing digest items with an unknown consensus engine will fail to
104    /// verify.
105    ///
106    /// Note that blocks must always contain digest items that are relevant to the current
107    /// consensus algorithm. This option controls what happens when blocks contain additional
108    /// digest items that aren't recognized by the implementation.
109    ///
110    /// Passing `true` can lead to blocks being considered as valid when they shouldn't, as these
111    /// additional digest items could have some logic attached to them that restricts which blocks
112    /// are valid and which are not.
113    ///
114    /// However, since a recognized consensus engine must always be present, both `true` and
115    /// `false` guarantee that the number of authorable blocks over the network is bounded.
116    pub allow_unknown_consensus_engines: bool,
117
118    /// Pre-allocated capacity for the number of block sources.
119    pub sources_capacity: usize,
120
121    /// Pre-allocated capacity for the number of blocks between the finalized block and the head
122    /// of the chain.
123    ///
124    /// Should be set to the maximum number of block between two consecutive justifications.
125    pub blocks_capacity: usize,
126
127    /// Maximum number of blocks of unknown ancestry to keep in memory. A good default is 1024.
128    ///
129    /// When a potential long fork is detected, its blocks are downloaded progressively in
130    /// descending order until a common ancestor is found.
131    /// Unfortunately, an attack could generate fake very long forks in order to make the node
132    /// consume a lot of memory keeping track of the blocks in that fork.
133    /// In order to avoid this, a limit is added to the number of blocks of unknown ancestry that
134    /// are kept in memory.
135    ///
136    /// Note that the download of long forks will always work no matter this limit. In the worst
137    /// case scenario, the same blocks will be downloaded multiple times. There is an implicit
138    /// minimum size equal to the number of sources that have been added to the state machine.
139    ///
140    /// Increasing this value has no drawback, except for increasing the maximum possible memory
141    /// consumption of this state machine.
142    //
143    // Implementation note: the size of `disjoint_headers` can temporarily grow above this limit
144    // due to the internal processing of the state machine.
145    pub max_disjoint_headers: usize,
146
147    /// Maximum number of simultaneous pending requests made towards the same block.
148    ///
149    /// Should be set according to the failure rate of requests. For example if requests have a
150    /// `10%` chance of failing, then setting to value to `2` gives a `1%` chance that downloading
151    /// this block will overall fail and has to be attempted again.
152    ///
153    /// Also keep in mind that sources might maliciously take a long time to answer requests. A
154    /// higher value makes it possible to reduce the risks of the syncing taking a long time
155    /// because of malicious sources.
156    ///
157    /// The higher the value, the more bandwidth is potentially wasted.
158    pub max_requests_per_block: NonZero<u32>,
159
160    /// If true, the body of a block is downloaded (if necessary) before a
161    /// [`ProcessOne::BlockVerify`] is generated.
162    pub download_bodies: bool,
163}

186struct Source<TSrc> {
187    /// Each source stores between zero and two finality proofs that haven't been verified yet.
188    ///
189    /// If more than two finality proofs are received from the same source, only the one with the
190    /// lowest target block and the one with the highest target block are kept in memory. This is
191    /// done in order to have a maximum bound to the amount of memory that is allocated per source
192    /// and avoid DoS attack vectors.
193    ///
194    /// The finality proof with the highest target block is the "best" finality proof. However,
195    /// keeping the finality proof with the lowest target block guarantees that, assuming the
196    /// source isn't malicious, we will able to make *some* progress in the finality.
197    unverified_finality_proofs: SourcePendingJustificationProofs,
198
199    /// Height of the highest finalized block according to that source. `0` if unknown.
200    finalized_block_number: u64,
201
202    /// Similar to [`Source::unverified_finality_proofs`]. Contains proofs that have been checked
203    /// and have been determined to not be verifiable right now.
204    pending_finality_proofs: SourcePendingJustificationProofs,
205
206    /// Opaque data chosen by the API user.
207    user_data: TSrc,
208}
209
210enum SourcePendingJustificationProofs {
211    None,
212    One {
213        target_height: u64,
214        proof: FinalityProofs,
215    },
216    Two {
217        low_target_height: u64,
218        low_proof: FinalityProofs,
219        high_target_height: u64,
220        high_proof: FinalityProofs,
221    },
222}
223
224impl SourcePendingJustificationProofs {
225    fn is_none(&self) -> bool {
226        matches!(self, SourcePendingJustificationProofs::None)
227    }
228
229    fn insert(&mut self, new_target_height: u64, new_proof: FinalityProofs) {
230        // An empty list of justifications is an invalid state.
231        debug_assert!(
232            !matches!(&new_proof, FinalityProofs::Justifications(list) if list.is_empty())
233        );
234
235        match mem::replace(self, SourcePendingJustificationProofs::None) {
236            SourcePendingJustificationProofs::None => {
237                *self = SourcePendingJustificationProofs::One {
238                    target_height: new_target_height,
239                    proof: new_proof,
240                };
241            }
242            SourcePendingJustificationProofs::One {
243                target_height,
244                proof,
245            } if target_height < new_target_height => {
246                *self = SourcePendingJustificationProofs::Two {
247                    low_target_height: target_height,
248                    low_proof: proof,
249                    high_target_height: new_target_height,
250                    high_proof: new_proof,
251                };
252            }
253            SourcePendingJustificationProofs::One {
254                target_height,
255                proof,
256            } if target_height > new_target_height => {
257                *self = SourcePendingJustificationProofs::Two {
258                    low_target_height: new_target_height,
259                    low_proof: new_proof,
260                    high_target_height: target_height,
261                    high_proof: proof,
262                };
263            }
264            SourcePendingJustificationProofs::One { .. } => {
265                *self = SourcePendingJustificationProofs::One {
266                    target_height: new_target_height,
267                    proof: new_proof,
268                };
269            }
270            SourcePendingJustificationProofs::Two {
271                high_target_height,
272                low_proof,
273                low_target_height,
274                ..
275            } if new_target_height >= high_target_height => {
276                *self = SourcePendingJustificationProofs::Two {
277                    high_proof: new_proof,
278                    high_target_height: new_target_height,
279                    low_proof,
280                    low_target_height,
281                };
282            }
283            SourcePendingJustificationProofs::Two {
284                high_proof,
285                high_target_height,
286                low_target_height,
287                ..
288            } if new_target_height <= low_target_height => {
289                *self = SourcePendingJustificationProofs::Two {
290                    high_proof,
291                    high_target_height,
292                    low_proof: new_proof,
293                    low_target_height: new_target_height,
294                };
295            }
296            val @ SourcePendingJustificationProofs::Two { .. } => {
297                *self = val;
298            }
299        }
300    }
301
302    fn take_one(&mut self) -> Option<FinalityProof> {
303        match mem::replace(self, SourcePendingJustificationProofs::None) {
304            SourcePendingJustificationProofs::None => {
305                *self = SourcePendingJustificationProofs::None;
306                None
307            }
308            SourcePendingJustificationProofs::One {
309                proof: FinalityProofs::GrandpaCommit(commit),
310                ..
311            } => {
312                *self = SourcePendingJustificationProofs::None;
313                Some(FinalityProof::GrandpaCommit(commit))
314            }
315            SourcePendingJustificationProofs::One {
316                proof: FinalityProofs::Justifications(justifications),
317                ..
318            } if justifications.len() == 1 => {
319                *self = SourcePendingJustificationProofs::None;
320                let j = justifications.into_iter().next().unwrap();
321                Some(FinalityProof::Justification(j))
322            }
323            SourcePendingJustificationProofs::One {
324                target_height,
325                proof: FinalityProofs::Justifications(mut justifications),
326            } => {
327                let j = justifications.pop().unwrap();
328                *self = SourcePendingJustificationProofs::One {
329                    target_height,
330                    proof: FinalityProofs::Justifications(justifications),
331                };
332                Some(FinalityProof::Justification(j))
333            }
334            SourcePendingJustificationProofs::Two {
335                high_proof: FinalityProofs::GrandpaCommit(commit),
336                low_proof,
337                low_target_height,
338                ..
339            } => {
340                *self = SourcePendingJustificationProofs::One {
341                    target_height: low_target_height,
342                    proof: low_proof,
343                };
344                Some(FinalityProof::GrandpaCommit(commit))
345            }
346            SourcePendingJustificationProofs::Two {
347                high_proof: FinalityProofs::Justifications(justifications),
348                low_proof,
349                low_target_height,
350                ..
351            } if justifications.len() == 1 => {
352                let j = justifications.into_iter().next().unwrap();
353                *self = SourcePendingJustificationProofs::One {
354                    target_height: low_target_height,
355                    proof: low_proof,
356                };
357                Some(FinalityProof::Justification(j))
358            }
359            SourcePendingJustificationProofs::Two {
360                high_proof: FinalityProofs::Justifications(mut justifications),
361                high_target_height,
362                low_proof,
363                low_target_height,
364            } => {
365                let j = justifications.pop().unwrap();
366                *self = SourcePendingJustificationProofs::Two {
367                    high_proof: FinalityProofs::Justifications(justifications),
368                    high_target_height,
369                    low_proof,
370                    low_target_height,
371                };
372                Some(FinalityProof::Justification(j))
373            }
374        }
375    }
376
377    fn merge(&mut self, other: Self) {
378        match other {
379            SourcePendingJustificationProofs::None => {}
380            SourcePendingJustificationProofs::One {
381                target_height,
382                proof,
383            } => self.insert(target_height, proof),
384            SourcePendingJustificationProofs::Two {
385                high_proof,
386                high_target_height,
387                low_proof,
388                low_target_height,
389            } => {
390                self.insert(high_target_height, high_proof);
391                self.insert(low_target_height, low_proof);
392            }
393        }
394    }
395}

503    /// Starts the process of inserting a new source in the [`AllForksSync`].
504    ///
505    /// This function doesn't modify the state machine, but only looks at the current state of the
506    /// block referenced by `best_block_number` and `best_block_hash`. It returns an enum that
507    /// allows performing the actual insertion.
508    pub fn prepare_add_source(
509        &'_ mut self,
510        best_block_number: u64,
511        best_block_hash: [u8; 32],
512    ) -> AddSource<'_, TBl, TRq, TSrc> {
513        if best_block_number <= self.chain.finalized_block_height() {
514            return AddSource::OldBestBlock(AddSourceOldBlock {
515                inner: self,
516                best_block_hash,
517                best_block_number,
518            });
519        }
520
521        let best_block_already_verified = self.chain.contains_non_finalized_block(&best_block_hash);
522        let best_block_in_disjoints_list = self
523            .inner
524            .blocks
525            .contains_unverified_block(best_block_number, &best_block_hash);
526
527        match (best_block_already_verified, best_block_in_disjoints_list) {
528            (false, false) => AddSource::UnknownBestBlock(AddSourceUnknown {
529                inner: self,
530                best_block_hash,
531                best_block_number,
532            }),
533            (true, false) => AddSource::BestBlockAlreadyVerified(AddSourceKnown {
534                inner: self,
535                best_block_hash,
536                best_block_number,
537            }),
538            (false, true) => AddSource::BestBlockPendingVerification(AddSourceKnown {
539                inner: self,
540                best_block_hash,
541                best_block_number,
542            }),
543            (true, true) => unreachable!(),
544        }
545    }

662    ///
663    /// This method doesn't modify the state machine in any way. [`AllForksSync::add_request`]
664    /// must be called in order for the request to actually be marked as started.
665    pub fn desired_requests(&self) -> impl Iterator<Item = (SourceId, &TSrc, RequestParams)> {
666        // Query justifications of blocks that are necessary in order for finality to progress
667        // against sources that have reported these blocks as finalized.
668        // TODO: make it clear in the API docs that justifications should be requested as part of a request
669        // TODO: this is O(n)
670        let justification_requests =
671            self.chain
672                .finality_checkpoints()
673                .flat_map(move |(block_height, block_hash)| {
674                    self.inner
675                        .blocks
676                        .sources()
677                        .filter(move |s| {
678                            // We assume that all sources have the same finalized blocks and thus
679                            // don't check hashes.
680                            self.inner.blocks[*s].unverified_finality_proofs.is_none()
681                                && self.inner.blocks[*s].finalized_block_number >= block_height
682                        })
683                        .map(move |source_id| {
684                            (
685                                source_id,
686                                &self.inner.blocks[source_id].user_data,
687                                RequestParams {
688                                    first_block_hash: *block_hash,
689                                    first_block_height: block_height,
690                                    num_blocks: NonZero::<u64>::new(1).unwrap(),
691                                },
692                            )
693                        })
694                });
695
696        let block_requests = self
697            .inner
698            .blocks
699            .desired_requests()
700            .filter(move |rq| {
701                !self
702                    .chain
703                    .contains_non_finalized_block(&rq.request_params.first_block_hash)
704            })
705            .map(move |rq| {
706                (
707                    rq.source_id,
708                    &self.inner.blocks[rq.source_id].user_data,
709                    rq.request_params,
710                )
711            });
712
713        justification_requests.chain(block_requests)
714    }

758    ///

765    /// the block with the hash that was referred by [`RequestParams::first_block_hash`]. Each

773        &'_ mut self,
774        request_id: RequestId,
775    ) -> (TRq, FinishRequest<'_, TBl, TRq, TSrc>) {
776        // Sets the `occupation` of `source_id` back to `AllSync`.
777        let (
778            pending_blocks::RequestParams {
779                first_block_hash: requested_block_hash,
780                first_block_height: requested_block_height,
781                ..
782            },
783            source_id,
784            request_user_data,
785        ) = self.inner.blocks.remove_request(request_id);
786
787        (
788            request_user_data,
789            FinishRequest {
790                inner: self,
791                source_id,
792                any_progress: false,
793                index_in_response: 0,
794                requested_block_hash,
795                requested_block_height,

802    /// Update the source with a newly-announced block.
803    ///
804    /// > **Note**: This information is normally reported by the source itself. In the case of a
805    /// >           a networking peer, call this when the source sent a block announce.
806    ///
807    /// # Panic
808    ///
809    /// Panics if `source_id` is invalid.
810    ///
811    pub fn block_announce(
812        &'_ mut self,
813        source_id: SourceId,
814        announced_scale_encoded_header: Vec<u8>,
815        is_best: bool,
816    ) -> BlockAnnounceOutcome<'_, TBl, TRq, TSrc> {
817        let announced_header = match header::decode(
818            &announced_scale_encoded_header,
819            self.chain.block_number_bytes(),
820        ) {
821            Ok(h) => h,
822            Err(error) => return BlockAnnounceOutcome::InvalidHeader(error),
823        };
824
825        let announced_header_number = announced_header.number;
826        let announced_header_parent_hash = *announced_header.parent_hash;
827        let announced_header_hash = announced_header.hash(self.chain.block_number_bytes());
828
829        // It is assumed that all sources will eventually agree on the same finalized chain. If
830        // the block number is lower or equal than the locally-finalized block number, it is
831        // assumed that this source is simply late compared to the local node, and that the block
832        // that has been received is either part of the finalized chain or belongs to a fork that
833        // will get discarded by this source in the future.
834        if announced_header_number <= self.chain.finalized_block_height() {
835            // Even if the block is below the finalized block, we still need to set it as the
836            // best block of this source, if anything for API consistency purposes.
837            if is_best {
838                self.inner.blocks.add_known_block_to_source_and_set_best(
839                    source_id,
840                    announced_header_number,
841                    announced_header_hash,
842                );
843            }
844
845            return BlockAnnounceOutcome::TooOld {
846                announce_block_height: announced_header_number,
847                finalized_block_height: self.chain.finalized_block_height(),
848            };
849        }
850
851        // If the block is already part of the local tree of blocks, nothing more to do.
852        if self
853            .chain
854            .contains_non_finalized_block(&announced_header_hash)
855        {
856            return BlockAnnounceOutcome::AlreadyVerified(AnnouncedBlockKnown {
857                inner: self,
858                announced_header_hash,
859                announced_header_number,
860                announced_header_parent_hash,
861                announced_header_encoded: announced_scale_encoded_header,
862                source_id,
863                is_in_chain: true,
864                is_best,
865            });
866        }
867
868        // At this point, we have excluded blocks that are already part of the chain or too old.
869        // We insert the block in the list of unverified blocks so as to treat all blocks the
870        // same.
871        if !self
872            .inner
873            .blocks
874            .contains_unverified_block(announced_header_number, &announced_header_hash)
875        {
876            BlockAnnounceOutcome::Unknown(AnnouncedBlockUnknown {
877                inner: self,
878                announced_header_hash,
879                announced_header_number,
880                announced_header_parent_hash,
881                announced_header_encoded: announced_scale_encoded_header,
882                source_id,
883                is_best,
884            })
885        } else {
886            BlockAnnounceOutcome::AlreadyPending(AnnouncedBlockKnown {
887                inner: self,
888                announced_header_hash,
889                announced_header_number,
890                announced_header_parent_hash,
891                announced_header_encoded: announced_scale_encoded_header,
892                is_in_chain: false,
893                source_id,
894                is_best,
895            })
896        }
897    }

915    /// Update the state machine with a Grandpa commit message received from the network.
916    ///
917    /// This function only inserts the commit message into the state machine, and does not
918    /// immediately verify it.
919    ///
920    /// # Panic
921    ///
922    /// Panics if `source_id` is invalid.
923    ///
924    pub fn grandpa_commit_message(
925        &mut self,
926        source_id: SourceId,
927        scale_encoded_commit: Vec<u8>,
928    ) -> GrandpaCommitMessageOutcome {
929        let source = &mut self.inner.blocks[source_id];
930
931        let block_number = match decode::decode_grandpa_commit(
932            &scale_encoded_commit,
933            self.chain.block_number_bytes(),
934        ) {
935            Ok(msg) => msg.target_number,
936            Err(_) => return GrandpaCommitMessageOutcome::ParseError,
937        };
938
939        // The finalized block number of the source is increased even if the commit message
940        // isn't known to be valid yet.
941        source.finalized_block_number = cmp::max(source.finalized_block_number, block_number);
942
943        source.unverified_finality_proofs.insert(
944            block_number,
945            FinalityProofs::GrandpaCommit(scale_encoded_commit),
946        );
947
948        GrandpaCommitMessageOutcome::Queued
949    }

951    /// Process the next block in the queue of verification.
952    ///
953    /// This method takes ownership of the [`AllForksSync`] and starts a verification
954    /// process. The [`AllForksSync`] is yielded back at the end of this process.
955    pub fn process_one(mut self) -> ProcessOne<TBl, TRq, TSrc> {
956        // Try to find a block to verify.
957        // All blocks are always verified before verifying justifications, in order to guarantee
958        // that the block that a justification targets has already been verified.
959        // TODO: revisit that ^ as advancing finality should have priority over advancing the chain
960        let block_to_verify = self.inner.blocks.unverified_leaves().find(|block| {
961            block.parent_block_hash == *self.chain.finalized_block_hash()
962                || self
963                    .chain
964                    .contains_non_finalized_block(&block.parent_block_hash)
965        });
966        if let Some(block) = block_to_verify {
967            return ProcessOne::BlockVerify(BlockVerify {
968                parent: self,
969                block_to_verify: block,
970            });
971        }
972
973        // Try to find a justification to verify.
974        // TODO: O(n)
975        let source_id_with_finality_proof = self
976            .inner
977            .blocks
978            .sources()
979            .find(|id| !self.inner.blocks[*id].unverified_finality_proofs.is_none());
980        if let Some(source_id_with_finality_proof) = source_id_with_finality_proof {
981            let finality_proof_to_verify = self.inner.blocks[source_id_with_finality_proof]
982                .unverified_finality_proofs
983                .take_one()
984                .unwrap(); // `take()` always returns `Some` because we've checked `is_none()` above
985            return ProcessOne::FinalityProofVerify(FinalityProofVerify {
986                parent: self,
987                source_id: source_id_with_finality_proof,
988                finality_proof_to_verify,
989            });
990        }
991
992        ProcessOne::AllSync { sync: self }
993    }

1044/// See [`AllForksSync::finish_request`].
1045pub struct FinishRequest<'a, TBl, TRq, TSrc> {
1046    inner: &'a mut AllForksSync<TBl, TRq, TSrc>,
1047
1048    /// Source that has sent the request that is being answered.
1049    source_id: SourceId,
1050
1051    /// Set to true if any block at all have been added.
1052    any_progress: bool,
1053
1054    /// Number of blocks added before through that data structure.
1055    index_in_response: usize,
1056
1057    /// Hash of the block that was initially request.
1058    requested_block_hash: [u8; 32],
1059    /// Height of the block that was initially request.
1060    requested_block_height: u64,
1061
1062    /// The next block to add should have a hash equal to this one.
1063    expected_next_hash: [u8; 32],
1064    /// The next block to add should have a height equal to this one.
1065    expected_next_height: u64,
1066}
1067
1068impl<'a, TBl, TRq, TSrc> FinishRequest<'a, TBl, TRq, TSrc> {
1069    /// Adds a block coming from the response that the source has provided.
1070    ///
1071    /// On success, the [`FinishRequest`] is turned into an [`AddBlock`]. The block is
1072    /// inserted in the state machine only after one of the methods in [`AddBlock`] is added.
1073    ///
1074    /// If an error is returned, the [`FinishRequest`] is turned back again into a
1075    /// [`AllForksSync`], but all the blocks that have already been added are retained.
1076    ///
1077    /// If [`Config::download_bodies`] was `false`, the content of `scale_encoded_extrinsics`
1078    /// is ignored.
1079    pub fn add_block(
1080        mut self,
1081        scale_encoded_header: Vec<u8>,
1082        scale_encoded_extrinsics: Vec<Vec<u8>>,
1083        scale_encoded_justifications: impl Iterator<Item = ([u8; 4], impl AsRef<[u8]>)>,
1084    ) -> Result<AddBlock<'a, TBl, TRq, TSrc>, AncestrySearchResponseError> {
1085        // Compare expected with actual hash.
1086        // This ensure that each header being processed is the parent of the previous one.
1087        if self.expected_next_hash != header::hash_from_scale_encoded_header(&scale_encoded_header)
1088        {
1089            return Err(AncestrySearchResponseError::UnexpectedBlock);
1090        }
1091
1092        // Invalid headers are erroneous.
1093        let decoded_header =
1094            match header::decode(&scale_encoded_header, self.inner.chain.block_number_bytes()) {
1095                Ok(h) => h,
1096                Err(err) => return Err(AncestrySearchResponseError::InvalidHeader(err)),
1097            };
1098
1099        // Also compare the block numbers.
1100        // The utility of checking the height (even though we've already checked the hash) is
1101        // questionable, but considering that blocks are identified with their combination of
1102        // hash and number, checking both the hash and number might prevent malicious sources
1103        // from introducing state inconsistenties, even though it's unclear how that could happen.
1104        if self.expected_next_height != decoded_header.number {
1105            return Err(AncestrySearchResponseError::UnexpectedBlock);
1106        }
1107
1108        // Check whether the SCALE-encoded extrinsics match the extrinsics root found in
1109        // the header.
1110        if self.inner.inner.blocks.downloading_bodies() {
1111            let calculated = header::extrinsics_root(&scale_encoded_extrinsics);
1112            if calculated != *decoded_header.extrinsics_root {
1113                return Err(AncestrySearchResponseError::ExtrinsicsRootMismatch);
1114            }
1115        }
1116
1117        // At this point, the source has given us correct blocks, and we consider the response
1118        // as a whole to be useful.
1119        self.any_progress = true;
1120
1121        // It is assumed that all sources will eventually agree on the same finalized chain. If
1122        // the block number is lower or equal than the locally-finalized block number, it is
1123        // assumed that this source is simply late compared to the local node, and that the block
1124        // that has been received is either part of the finalized chain or belongs to a fork that
1125        // will get discarded by this source in the future.
1126        if decoded_header.number <= self.inner.chain.finalized_block_height() {
1127            return Err(AncestrySearchResponseError::TooOld);
1128        }
1129
1130        // Convert the justifications in an "owned" format, because we're likely going to store
1131        // them.
1132        let justifications = scale_encoded_justifications
1133            .map(|(e, j)| (e, j.as_ref().to_owned()))
1134            .collect::<Vec<_>>();
1135
1136        // If the block is already part of the local tree of blocks, nothing more to do.
1137        // Note that the block body is silently discarded, as in the API only non-verified blocks
1138        // exhibit a body.
1139        if self
1140            .inner
1141            .chain
1142            .contains_non_finalized_block(&self.expected_next_hash)
1143        {
1144            if !justifications.is_empty() {
1145                self.inner.inner.blocks[self.source_id]
1146                    .unverified_finality_proofs
1147                    .insert(
1148                        decoded_header.number,
1149                        FinalityProofs::Justifications(justifications),
1150                    );
1151            }
1152
1153            return Ok(AddBlock::AlreadyInChain(AddBlockOccupied {
1154                inner: self,
1155                block_number: decoded_header.number,
1156                block_parent_hash: *decoded_header.parent_hash,
1157                block_header: scale_encoded_header,
1158                is_verified: true,
1159            }));
1160        }
1161
1162        // Block is not part of the finalized chain.
1163        // TODO: also give possibility to update user data
1164        if decoded_header.number == self.inner.chain.finalized_block_height() + 1
1165            && *decoded_header.parent_hash != *self.inner.chain.finalized_block_hash()
1166        {
1167            // TODO: remove_verify_failed
1168            // Block isn't part of the finalized chain.
1169            // This doesn't necessarily mean that the source and the local node disagree
1170            // on the finalized chain. It is possible that the finalized block has been
1171            // updated between the moment the request was emitted and the moment the
1172            // response is received.
1173            let error = AncestrySearchResponseError::NotFinalizedChain {
1174                discarded_unverified_block_headers: Vec::new(), // TODO: not properly implemented /!\
1175            };
1176            return Err(error);
1177        }
1178
1179        // At this point, we have excluded blocks that are already part of the chain or too old.
1180        // We insert the block in the list of unverified blocks so as to treat all blocks the
1181        // same.
1182        if !self
1183            .inner
1184            .inner
1185            .blocks
1186            .contains_unverified_block(decoded_header.number, &self.expected_next_hash)
1187        {
1188            Ok(AddBlock::UnknownBlock(AddBlockVacant {
1189                inner: self,
1190                block_number: decoded_header.number,
1191                block_parent_hash: *decoded_header.parent_hash,
1192                block_header: scale_encoded_header,
1193                scale_encoded_extrinsics,
1194                justifications,
1195            }))
1196        } else {
1197            if !justifications.is_empty() {
1198                self.inner.inner.blocks[self.source_id]
1199                    .unverified_finality_proofs
1200                    .insert(
1201                        decoded_header.number,
1202                        FinalityProofs::Justifications(justifications),
1203                    );
1204            }
1205
1206            if self.inner.inner.blocks.downloading_bodies() {
1207                self.inner
1208                    .inner
1209                    .blocks
1210                    .set_unverified_block_header_body_known(
1211                        decoded_header.number,
1212                        &self.expected_next_hash,
1213                        *decoded_header.parent_hash,
1214                    );
1215                self.inner
1216                    .inner
1217                    .blocks
1218                    .unverified_block_user_data_mut(decoded_header.number, &self.expected_next_hash)
1219                    .body = Some(scale_encoded_extrinsics);
1220            }
1221
1222            Ok(AddBlock::AlreadyPending(AddBlockOccupied {
1223                inner: self,
1224                block_number: decoded_header.number,
1225                block_parent_hash: *decoded_header.parent_hash,
1226                block_header: scale_encoded_header,
1227                is_verified: false,
1228            }))
1229        }
1230    }
1231
1232    /// Notifies of the end of the response, and returns back the [`AllForksSync`].
1233    ///
1234    /// It is legal to insert fewer blocks than the number of blocks that were requested through
1235    /// [`RequestParams::num_blocks`].
1236    /// However, if no block has been added at all (i.e. the response is empty), then the source
1237    /// of the request is marked as bad.
1238    ///
1239    /// > **Note**: Network protocols have a limit to the size of their response, meaning that all
1240    /// >           the requested blocks might not fit in a single response. For this reason, it
1241    /// >           is legal for a response to be shorter than expected.
1242    pub fn finish(self) {
1243        drop(self);
1244    }
1245}

1247impl<'a, TBl, TRq, TSrc> Drop for FinishRequest<'a, TBl, TRq, TSrc> {
1248    fn drop(&mut self) {
1249        // If this is reached, then none of the blocks the source has sent back were useful.
1250        if !self.any_progress {
1251            // Assume that the source doesn't know this block, as it is apparently unable to
1252            // serve it anyway. This avoids sending the same request to the same source over and
1253            // over again.
1254            self.inner.inner.blocks.remove_known_block_of_source(
1255                self.source_id,
1256                self.requested_block_height,
1257                &self.requested_block_hash,
1258            );
1259        }
1260    }
1261}

1263/// Result of calling [`FinishRequest::add_block`].

1939    /// Perform the verification.
1940    pub fn verify_header(
1941        mut self,
1942        now_from_unix_epoch: Duration,
1943    ) -> HeaderVerifyOutcome<TBl, TRq, TSrc> {
1944        let to_verify_scale_encoded_header = self.scale_encoded_header().to_owned(); // TODO: overhead
1945
1946        let result = match self
1947            .parent
1948            .chain
1949            .verify_header(to_verify_scale_encoded_header, now_from_unix_epoch)

1951            Ok(blocks_tree::HeaderVerifySuccess::Verified {

1969                self.parent.inner.blocks.mark_unverified_block_as_bad(
1970                    self.block_to_verify.block_number,
1971                    &self.block_to_verify.block_hash,
1972                );
1973
1974                Err(HeaderVerifyError::ConsensusMismatch)
1975            }
1976            Err(blocks_tree::HeaderVerifyError::UnknownConsensusEngine) => {
1977                // Remove the block from `pending_blocks`.
1978                self.parent.inner.blocks.mark_unverified_block_as_bad(
1979                    self.block_to_verify.block_number,
1980                    &self.block_to_verify.block_hash,
1981                );
1982
1983                Err(HeaderVerifyError::UnknownConsensusEngine)
1984            }
1985            Ok(blocks_tree::HeaderVerifySuccess::Duplicate)
1986            | Err(
1987                blocks_tree::HeaderVerifyError::BadParent { .. }
1988                | blocks_tree::HeaderVerifyError::InvalidHeader(_),
1989            ) => unreachable!(),
1990        };
1991
1992        match result {
1993            Ok((verified_header, is_new_best)) => HeaderVerifyOutcome::Success {
1994                is_new_best,
1995                success: HeaderVerifySuccess {
1996                    parent: self.parent,
1997                    block_to_verify: self.block_to_verify,
1998                    verified_header,
1999                },
2000            },
2001            Err(error) => HeaderVerifyOutcome::Error {
2002                sync: self.parent,
2003                error,
2004            },
2005        }
2006    }

2011    }

2137///
2138/// Internally holds the [`AllForksSync`].

2140    parent: AllForksSync<TBl, TRq, TSrc>,
2141    /// Source that has sent the finality proof.
2142    source_id: SourceId,
2143    /// Justification and its consensus engine id, or commit that can be verified.

2145}
2146
2147impl<TBl, TRq, TSrc> FinalityProofVerify<TBl, TRq, TSrc> {
2148    /// Returns the source the justification was obtained from.
2149    pub fn sender(&self) -> (SourceId, &TSrc) {
2150        (self.source_id, &self.parent[self.source_id])
2151    }
2152
2153    /// Perform the verification.
2154    ///
2155    /// A randomness seed must be provided and will be used during the verification. Note that the
2156    /// verification is nonetheless deterministic.
2157    pub fn perform(
2158        mut self,
2159        randomness_seed: [u8; 32],
2160    ) -> (
2161        AllForksSync<TBl, TRq, TSrc>,
2162        FinalityProofVerifyOutcome<TBl>,
2163    ) {
2164        let finality_apply = match self.finality_proof_to_verify {
2165            FinalityProof::GrandpaCommit(scale_encoded_commit) => {
2166                match self
2167                    .parent
2168                    .chain
2169                    .verify_grandpa_commit_message(&scale_encoded_commit, randomness_seed)
2170                {
2171                    Ok(finality_apply) => finality_apply,
2172
2173                    // In case where the commit message concerns a block older or equal to the
2174                    // finalized block, the operation is silently considered successful.
2175                    Err(blocks_tree::CommitVerifyError::FinalityVerify(
2176                        blocks_tree::FinalityVerifyError::EqualToFinalized
2177                        | blocks_tree::FinalityVerifyError::BelowFinalized,
2178                    )) => return (self.parent, FinalityProofVerifyOutcome::AlreadyFinalized),
2179
2180                    // The commit can't be verified yet.
2181                    Err(
2182                        blocks_tree::CommitVerifyError::FinalityVerify(
2183                            blocks_tree::FinalityVerifyError::UnknownTargetBlock {
2184                                block_number,
2185                                ..
2186                            },
2187                        )
2188                        | blocks_tree::CommitVerifyError::FinalityVerify(
2189                            blocks_tree::FinalityVerifyError::TooFarAhead {
2190                                justification_block_number: block_number,
2191                                ..
2192                            },
2193                        )
2194                        | blocks_tree::CommitVerifyError::NotEnoughKnownBlocks {
2195                            target_block_number: block_number,
2196                        },
2197                    ) => {
2198                        self.parent.inner.blocks[self.source_id]
2199                            .pending_finality_proofs
2200                            .insert(
2201                                block_number,
2202                                FinalityProofs::GrandpaCommit(scale_encoded_commit),
2203                            );
2204                        return (
2205                            self.parent,
2206                            FinalityProofVerifyOutcome::GrandpaCommitPending,
2207                        );
2208                    }
2209
2210                    // Any other error means that the commit is invalid.
2211                    Err(err) => {
2212                        return (
2213                            self.parent,
2214                            FinalityProofVerifyOutcome::GrandpaCommitError(err),
2215                        );
2216                    }
2217                }
2218            }
2219
2220            FinalityProof::Justification((consensus_engine_id, scale_encoded_justification)) => {
2221                match self.parent.chain.verify_justification(
2222                    consensus_engine_id,
2223                    &scale_encoded_justification,
2224                    randomness_seed,
2225                ) {
2226                    Ok(finality_apply) => finality_apply,
2227
2228                    // In case where the commit message concerns a block older or equal to the
2229                    // finalized block, the operation is silently considered successful.
2230                    Err(blocks_tree::JustificationVerifyError::FinalityVerify(
2231                        blocks_tree::FinalityVerifyError::EqualToFinalized
2232                        | blocks_tree::FinalityVerifyError::BelowFinalized,
2233                    )) => return (self.parent, FinalityProofVerifyOutcome::AlreadyFinalized),
2234
2235                    // Note that, contrary to commits, there's no such thing as a justification
2236                    // that can't be verified yet.
2237                    Err(err) => {
2238                        return (
2239                            self.parent,
2240                            FinalityProofVerifyOutcome::JustificationError(err),
2241                        );
2242                    }
2243                }
2244            }
2245        };
2246
2247        // Commit or justification successfully verified.
2248        // Update the local state with the newly-finalized block.
2249
2250        let finalized_blocks_iter = finality_apply.apply();
2251        let updates_best_block = finalized_blocks_iter.updates_best_block();
2252        let mut finalized_blocks = Vec::new();
2253        let mut pruned_blocks = Vec::new();
2254        // TODO: a bit weird to perform a conversion here
2255        for block in finalized_blocks_iter {
2256            if matches!(block.ty, blocks_tree::RemovedBlockType::Finalized) {
2257                finalized_blocks.push(RemovedBlock {
2258                    block_hash: block.block_hash,
2259                    block_number: block.block_number,
2260                    user_data: block.user_data,
2261                    scale_encoded_header: block.scale_encoded_header,
2262                });
2263            } else {
2264                pruned_blocks.push(RemovedBlock {
2265                    block_hash: block.block_hash,
2266                    block_number: block.block_number,
2267                    user_data: block.user_data,
2268                    scale_encoded_header: block.scale_encoded_header,
2269                });
2270            }
2271        }
2272        let _finalized_blocks = self
2273            .parent
2274            .inner
2275            .blocks
2276            .set_finalized_block_height(finalized_blocks.last().unwrap().block_number);
2277
2278        (
2279            self.parent,
2280            FinalityProofVerifyOutcome::NewFinalized {
2281                finalized_blocks_newest_to_oldest: finalized_blocks,
2282                pruned_blocks,
2283                updates_best_block,
2284            },
2285        )
2286    }

2290        self.parent
2291    }
