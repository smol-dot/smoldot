pub fn finish_request( &mut self, request_id: RequestId, ) -> (TRq, FinishRequest<'_, TBl, TRq, TSrc>)

Call in response to a request being successful or failing.

This state machine doesnโt differentiate between successful or failed requests. If a request has failed, call this function and immediately call FinishRequest::finish. Additionally, it is allow to insert fewer blocks than the number indicated in RequestParams::num_blocks.

The added blocks are expected to be sorted in decreasing order. The first block should be the block with the hash that was referred by RequestParams::first_block_hash. Each subsequent element is then expected to be the parent of the previous one.
